Overview

Having a FileMaker Pro relational database with MARC records from 1978 - 2013 pulled into a relational database format, and several Excel files of aggregated data from that database, I spent these few days looking for patterns. The goals were to answer some questions and/or pull out interesting trends and notes about the data available, particularly in reference to the dates of a few relevant cases.

The results of the week at a high level consist of:

  • a couple datasets corrected for potential confusions and/or with additional columns of information (all documented below)
  • a few plots of information over time in addition to the existing charts in the Excel sheets
  • a few statistical results to answer some questions
  • a few statistical/analysis results to suggest possible further analysis

Final Dataset

Takeaway Points:

  • There is a significant increase in % of registered works in Security Interest documents after the 2002 Ninth Circuit case limiting the 1990 Peregrine decision, but not for Assignment documents.
  • There are several strategies that were applied in my work this week and that could be applied in future analysis for continued cleaning and visualizing of data; the clearest representation and visualization of these datasets seems to be in time series graphs, whether in Excel or in other software (like R).
  • It is possible that three clusters of data represent the variety of information presented in this data best, but further analysis is required for confirmation as well as for definition of precisely where the data should be 'binned' with regard to number of works per document.

All are discussed in more detail below.

II. Data cleaning and preparation

There were a couple issues with the original data to address, which I dealt with before proceeding with analysis. Those issues and how I dealt with them follow:

Separating out relevant types of documents

Though there are many (28) coded document types available in the database I was working off of, there were three with which I was particularly concerned, which could overall be divided into two more general types:

  • Assignments (coded type 10)
  • Security Interests, comprised of Grants of Security Interest and Releases of Security Interests (coded as types 13 and 14 respectively)

To pull out only the two and reassign them from their coded values to their names, I wrote a function to quickly go through the vector (basically a 'column' of data in R) and assign each value as appropriate:

whichtype <- function(x) { 
  if (x == '10' | x == 10) {
    return ("Assignment");
  } else {
    return ("SecurityInterest");
  }
}

I did this because, with only two options for the document type, the names were easier to read and manipulate in R data frames (which are viewable much like Excel spreadsheets are) and would be just as easy to analyze. It was also easier to assign a single category to Security Interests at once rather than repeatedly referring both to Grants and Releases.

After loading in data:

setwd("~/Dropbox/CopyrightData-ASB-JZC"); # for my local filesystem, this should point to the directory where relevant data files are saved, and is here in order to load data for report displays
drd <- read.csv("DocumentsData_wRegNum.txt",sep="\t"); # saved excel sheet in tab-delimited .txt for analysis, opening that file here

I did a bit more processing, intermediate pieces in order to get the data in a very easily usable format for analysis. Below is code, commented with what each line does, all of which together which does this intermediate processing.

dr <- subset(drd, drd$Coded.Doc.Type == '10'); # assignments

Years whose data was thrown off by a single large document

I did some tests as to whether there was a statistically significant difference between the numbers of registration numbers corresponding to works in each document before or after the September 11, 2002 Ninth Circuit Court case which limited the Peregrine (1990) case to copyrighted works that had been registered. (Analysis below.)

Looking at the tabulated data, it was noticed that there were a few years whose percentages of registered works were surprisingly low, and we suspected that one document with many works, 0 of which had registrations, may have been throwing off the percentages notably.

On examination, these two documents were identifiable, and with the accurate information I could go into the dataset and make the manual change. (Because this change was manual, it is reflected in the dataset I've loaded in above and noted here for the sake of noting all pieces of this week's process.)

Blank document type fields

Because these records were in many cases transcribed from paper records, there were many whose Document Type field was blank. Some analysis done involved aggregating the works/document by year. (Analysis again detailed below.)

However, in order to include these documents as with the type field left blank and no note that they are identified as something unusual, I loaded in the data and then created two new columns: AdjAssignments and AdjSecurityInterests. These can be found in the final dataset Excel workbook.

The formula to reassign documents with blank Type fields:

  • multiply the number of documents with blank type fields in a given year by (that year's total Assignments / that year's total recorded documents)
  • multiply the number of documents with blank type fields in a given year by (that year's total Security Interests / that year's total recorded documents)
  • add the results of these calculations to the values of total Assignments for that year and total Security Interests for that year, respectively

Essentially, the AdjAssignments and AdjSecurityInterests columns reflect a percentage of the documents with blank type field assigned to the Assignment or Security Interest type category proportionally to the distribution of total recorded documents of these two types, when looking at a collection of documents which are Assignments, which are Security Interests, and documents with unrecorded (blank) types.

For more fine-grained analysis, one might consider assigning the percentages proportional to total documents in the original dataset (that is: if Assignments were 35% of the entire dataset and Security Interests were 46%, one could assign 35% of the blank-type documents to the Assignments category…). Whether this is a good idea for analysis may depend on other situational factors/other knowledge about this dataset.

The R code to load in this year-aggregated dataset and create these adjusted columns is as follows:


# take a look at works aggregated by year and numbers of respective categories
# first, read in data with numbers of blanks document types/yr and unidentified (continuing process) document types/yr
dbl <- read.csv("Docs2w500As.txt",sep="\t");
rownames(dbl) = dbl$Year; # make row names the years in the Year column of the dataframe
# create two new vectors for adjusted assignments and security interests, assigning proportionate #s of assignments/security interests from blanks
adjAssignments <- round(as.numeric(dbl$Blank.500a) * (as.numeric(dbl$Assignments)/as.numeric(dbl$Documents.Recorded)),digits=0) + dbl$Assignments;
adjSecInts <- round(as.numeric(dbl$Blank.500a) * (as.numeric(dbl$SecurityInterestsTotal)/as.numeric(dbl$Documents.Recorded)),digits=0) + dbl$SecurityInterestsTotal;
# add those columns to the aggregated dataframe (here, year aggregation was already completed in Excel, so here I am loading that in as-is and adding these calculations for adjusted column values)
data.corrfor.blanks <- data.frame(dbl,AdjAssignments=adjAssignments, AdjSecurityInterests=adjSecInts);
# take away cols for better clarity
data.corrfor.blanks <- data.corrfor.blanks[,c(1:4,6:10,14:16)];

III. Analysis and Results

There are three concrete deliverables that resulted from this analysis, and a set of suggestions that may lead to further research/analysis regarding clustering. They are as follows, in detail (with R code used to perform the March 2014 analyses included).

Examining average number of works per document with multiple works by document type and by year, 1978-2013

Given the data detailed above, I knew the numbers of works in documents with multiple works was something that might be interesting to examine. Many documents are made up of just one single work, but there could be interesting patterns to be found among documents with more than 1 single work associated with them (much prior analysis having been done to determine how many works belonged to each document).

As such, I created a subset of the data that consisted of documents made up of more than 1 single work:

mult_works_data <- subset(all.relevant.regnums, all.relevant.regnums$WorksCount > 1); # all documents with more than 1 work

I could then aggregate these numbers by year:

mwd.byyear <- aggregate(cbind(mult_works_data$WorksCount), by=list(mult_works_data$DocType,substr(as.character(mult_works_data$date),1,4)),FUN=mean); # get mean works/document by year
mwd.byyear <- data.frame(NamedDocType=mwd.byyear$Group.1,Year=mwd.byyear$Group.2,AvgWorksCount=mwd.byyear$V1); # correct column names for clarity

I did not aggregate the number of registered works, because that seemed to be a separate piece of analysis from the examination of how many works there were for each document, and it did not seem to make sense here as a metric to be summed or averaged by document type and/or year.

Having these numbers of documents with multiple works in them, however, allowed me to plot them for a clearer examination of works per document in the cases where a document had multiple works (the unusual cases overall):

# creates plot of average works/document by year
qplot(mwd.byyear$Year,mwd.byyear$AvgWorksCount,geom=c("jitter"),colour=mwd.byyear$NamedDocType,shape=mwd.byyear$DocType,ylab="Average Number of Works Per Document",xlab="Year",main="Average number of works per document, by document type, 1973-2013")+theme(axis.text.x = element_text(angle = 45, hjust = 1));

Testing for statistical significance of difference in registered works post-2002 case

On September 11th, 2002, the Ninth Circuit Court ruled such that the 1990 Peregrine case was limited. I hoped to get an idea of whether the number of works with registration numbers increased among Security Interest documents (grants or releases, since these were aggregated into one category, “Security Interests”) due to this ruling.

First, I went back to the original dataset and separated out the set of (all) documents into those recorded before September 11, 2002, and those recorded after:

all.regnums.post2002 <- subset(all.relevant.regnums,date > as.Date("2002-09-11"));
all.regnums.pre2002 <- subset(all.relevant.regnums, date < as.Date("2002-09-11"));

Then I aggregated each of these subsets by year, so the data would be easier to view and plot. (Note that there is a 2002 category in each subset, because the case happened mid-2002.)

## post 2002 case set
ar.post2002.byyear <- aggregate(cbind(all.regnums.post2002$RegWorksCount, all.regnums.post2002$WorksCount),by=list(all.regnums.post2002$DocType,substr(as.character(all.regnums.post2002$date),1,4)),FUN=sum);
# clean up column names
ar.post2002.byyear <- data.frame(Year=ar.post2002.byyear$Group.2,DocType=ar.post2002.byyear$Group.1,RegWorksCount=ar.post2002.byyear$V1,WorksCount=ar.post2002.byyear$V2);
## pre 2002 case set
ar.pre2002.byyear <- aggregate(cbind(all.regnums.pre2002$RegWorksCount, all.regnums.pre2002$WorksCount),by=list(all.regnums.pre2002$DocType,substr(as.character(all.regnums.pre2002$date),1,4)),FUN=sum);
# clean up column names
ar.pre2002.byyear <- data.frame(Year=ar.pre2002.byyear$Group.2,DocType=ar.pre2002.byyear$Group.1,RegWorksCount=ar.pre2002.byyear$V1,WorksCount=ar.pre2002.byyear$V2);

I added a column to each of these data frames which represented the average percentage of works with registration numbers, so we could examine average percentage of works with registration numbers per year, before and after the 2002 case.

# So add a column for %s in each case
percRegWorks.postcase <- as.numeric(ar.post2002.byyear$RegWorksCount)/as.numeric(ar.post2002.byyear$WorksCount);
percRegWorks.precase <- as.numeric(ar.pre2002.byyear$RegWorksCount)/as.numeric(ar.pre2002.byyear$WorksCount);
# add these columns to the respective dataframes, so these are the full datasets
ar.post2002.byyear <- data.frame(ar.post2002.byyear,PercentWorksRegNums=percRegWorks.postcase);
ar.pre2002.byyear <- data.frame(ar.pre2002.byyear,PercentWorksRegNums=percRegWorks.precase);

Finally, I extracted subsets of each of these (pre- and post- the 2002 case) that consisted of only Security Interest documents.

# let's take a look at just security interests from before and after that case
secint.post02 <- subset(ar.post2002.byyear, ar.post2002.byyear$DocType == "SecurityInterest");
secint.pre02 <- subset(ar.pre2002.byyear, ar.pre2002.byyear$DocType == "SecurityInterest");

Now, we have two datasets to look at. Security Interest documents from before and after the 2002 case limiting the Peregrine decision, each of which contains information about the average percentage of works in each document which have registration numbers per year. (It is possible that a single document, despite being an outlier, could notably affect this data; this led to the fix mentioned in I(2), “Years Whose Data was Thrown Off by a Single Large Document.”)

Since we want to get an idea of whether the 2002 case affected the percentage of works with registration numbers, a good first step is to see whether the mean average percentage (that's the mean of all average (mean) percentages per year) was greater after the 2002 case in a statistically significant amount. To do this, we perform an unpaired t-test, like so, and print the result for evaluation.

print(t.test(secint.post02$PercentWorksRegNums,secint.pre02$PercentWorksRegNums,alternative=c("greater"))); # runs unpaired t-test, to see whether the mean of % of security interest works with registration numbers is significantly greater after 2002 case than before 

## 
##  Welch Two Sample t-test
## 
## data:  secint.post02$PercentWorksRegNums and secint.pre02$PercentWorksRegNums 
## t = 5.355, df = 34.45, p-value = 2.865e-06
## alternative hypothesis: true difference in means is greater than 0 
## 95 percent confidence interval:
##  0.1933    Inf 
## sample estimates:
## mean of x mean of y 
##    0.7351    0.4527

As you can see in the results, there is a p-value of 2.865e-06, much less than the 0.05 significance level I've chosen to use in this report. This means that the alternative hypothesis is supported – it is very unlikely to be the case by chance – and the mean percent of works with registration numbers in Security Interest documents after September 11, 2002 is statistically significantly greater than before September 11, 2002. In other words, there is a significant difference in the percentage of works with registration numbers in Security Interest documents before and after the 2002 Ninth Circuit case.

Next, we want to examine the same for documents of type Assignment: if the same holds true for Assignments, since those documents were not affected by the September 2002 case, it may be that the 2002 case did not actually have an effect on the percent of registered works per document, and would suggest that much more analysis is necessary. Thus,

Separation of Assignment type documents from before and after the 2002 case, just like was done above for Security Interest documents.

# just assignments
assigns.post02 <- subset(ar.post2002.byyear, ar.post2002.byyear$DocType == "Assignment");
assigns.pre02 <- subset(ar.pre2002.byyear, ar.pre2002.byyear$DocType == "Assignment");

Then, running and printing results of the same t-test on these datasets:

print(t.test(assigns.post02$PercentWorksRegNums,assigns.pre02$PercentWorksRegNums,alternative=c("greater")));

## 
##  Welch Two Sample t-test
## 
## data:  assigns.post02$PercentWorksRegNums and assigns.pre02$PercentWorksRegNums 
## t = 0.1419, df = 26.5, p-value = 0.4441
## alternative hypothesis: true difference in means is greater than 0 
## 95 percent confidence interval:
##  -0.08671      Inf 
## sample estimates:
## mean of x mean of y 
##    0.3569    0.3490

The p-value of this test is 0.4441 – far above the 0.05 significance level, which suggests there is no significant difference in percentage of registered works per document for documents that are assignments.

Though further analysis may be necessary, these preliminary results suggest that the Ninth Circuit Case of September 2002 which limited the 1990 Peregrine decision may have had the effect of significantly increasing the number of works registered in Security Interest Grants and Security Interest Releases.

Potential data clustering

Most documents in this overall data set of recorded documents since about 1978 have only one work per document. However, of those with multiple works in a single document (separated out into a subset above, see mult_works_data), it seemed useful to discover the best way to bin the data such that the most information could be gained from it, since there is a big variety in nubers of works per document where the number > 1. For example, should documents with 2-20 works be separated from documents with 20 - 200, and them from those with 200-1000 works per document..? What separations and comparisons will provide the most useful analysis?

R can help perform clustering algorithms on datasets; it offers several options. More analysis needs to be done to determine the best bins for the data. Through some preliminary k-means clustering analysis, with the R function pam (“robust k-means clustering”) in the cluster library, I determined that 3 might be a good number of divisions for the multi-works documents data, though again, further analysis is required to confirm. (See the Further analysis for slightly more discussion.)

IV. Discussion and further analysis

The analysis done this week leads me to believe there is more valuable analysis that might be done on these datasets to identify interesting patterns and learn more about U.S. copyright history, especially improved with more domain knowledge. In particular, I see a use for a few types of additional analysis:

  • examination of further variables for means-comparison of certain factors before and after certain rulings (this would require more domain knowledge, of course)

  • continued examinination of clustering algorithms for this data to determine the salient amount of clusters that seem to exist among this dataset across several axes. Principal Component Analysis may also be a useful means of analysis in order to identify some primary variables of interest for e.g. clustering analysis, but an accurate job would take a bit more time.

  • finally, with regard to the question of whether one document with many works, none of which were recorded, throwing off a given year's percentage of works with registration numbers (in part addressed earlier, in the Data Preparation section above), a modified Thompson-Tau test may be useful to identify outliers, which could then be examined more closely by domain experts.
