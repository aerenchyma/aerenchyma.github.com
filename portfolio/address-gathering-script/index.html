Address-gathering Ruby script(s) for UWSEM

A friend at United Way for SouthEastern Michigan was working on a mapping project with ArcGIS for which, among other things, she needed a dataset of the addresses of places of worship within the tri-county area in Michigan. However, this isn’t a common type of publicly available data: mosques, churches, temples of different religions and communities were listed publicly and/or kept in public databases, but rarely were these things aggregated for easy use, especially not in a format she could use.

Humans are good at this kind of problem: knowing what to search, and how to tell a result is good. The project wasn’t big enough at the time to address the second part computationally, but the first part had a time estimate of months which this set of scripts reduced to days. Written in Ruby, it allows my friend (or whomever) to choose a search term a region within which to search. It returns CSV files of this already-public but difficult-to-access data of lengths that were workable with ArcGIS + a middling connection (in this case, 1000 lines), waiting long enough between hits that it causes no server difficulty (when scraping: be polite). Another, smaller script is included to allow someone not quite fluent in Ruby/Python/etc or Excel to filter out these entries by various criteria.

Another benefit of this small project was that it was a friend’s first introduction to scripting, and prompted her to engage differently with mapping software her organization dealt with and to begin learning Python.

The code can be found on GitHub here. I wrote about an earlier iteration of this project (later parts of the story to come eventually) here.
